Download OpenAPI specification:

V2C Support: info@v2charge.com

Introduction

This is the V2C RESTful API (Application Programming Interface), which allows for interaction with V2C's RESTful web services. It enables controlling charging devices, managing RFID cards, retrieving statistics, and configuring specific device parameters.

With this API, you can use your e-Charger through applications or services you create yourself, without needing the official apps. This allows you to create automations or interesting services for your chargers through programming.

Authentication

The API uses a Token (API Key) for authentication. You must provide your personal token in the apikey header of every request. If the apikey is invalid or not provided, the API will return a 401 Unauthorized error.

How to obtain your API Key

To obtain your authentication token, you can go to v2c.cloud, select the API section in the left-hand side menu, and generate it.

For more information, visit: https://v2charge.com/support/v2c-cloud/get-token-id-api-rest-cloud/

Limitations (Rate Limit)

Each user has a limit of 1000 daily calls (requests). If you require more calls, please contact support.

All responses will include the following headers to help you monitor your usage:

  • RateLimit-Limit: The maximum number of requests allowed in the daily period (e.g., 1000).
  • RateLimit-Remaining: The number of requests remaining in the current daily period.
